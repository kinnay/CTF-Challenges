
This directory contains the source files of the challenge. The following files can be found here:

* `/assets`: the PNG images from which the `.gtx` files were generated
* `/fs`: this is the root of the filesystem that is mounted on the Wii U
    * `/code`: this folder contains the main executable of the game during building
    * `/content`: this folder contains the data files that are shipped with the game
        * `/shaders`: a basic shader that is used for rendering (this one comes from the Rio framework)
        * `/textures`: the `.gtx` files that were generated from the assets folder
* `/source`: the C++ source code of the game
* `/shader`:
    * `/verifier.cs`: the source code of the compute shader that verifies the player input and generates the flag or try harder message
    * `/verifier.dump`: a shader assembly dump that was generated by the shader compiler
    * `/verifier.gsh`: the compiled compute shader that is embedded in the main executable
* `/convert_textures.py`: a script that generates the `.gtx` files from the assets folder using `TexConv2.exe`.
* `/create_binary_blob.py`: the script that generates the flag verification matrix and the binary blob that is passed to the shader for verification

Notes:
* The assets come from https://kenney.nl and https://roastedcodestudio.itch.io/balloon-pop-asset.
* The [rio framework by aboood40091](https://github.com/aboood40091/rio) was used to program the game.
